# Source of Truth Policy

## Goal
Keep recommendations anchored to authoritative sources instead of drifting toward stale community folklore.

## Priority Order
1. Apple official documentation
2. Swift.org official documentation and proposals when relevant
3. Apple WWDC sessions and official sample code
4. Official documentation / release notes / migration guides for third-party libraries that are already under evaluation

## Default Rule
Do **not** use random blogs, tutorials, or forum posts as the primary basis for stable recommendations.

## Allowed Use of Non-Official Sources
Non-official sources may help discover questions, but they must not become the final authority for baseline recommendations.

## Review Requirement
Any recommendation that diverges from the official baseline must explain:
- why the official path is insufficient here,
- which project constraints justify the divergence,
- what migration or lock-in cost that divergence creates.
