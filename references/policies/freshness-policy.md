# Freshness Policy

## Goal
Keep the plugin aligned with official documentation over time.

## Review Cadence
- high-priority topics: every 30 days
- medium-priority topics: every 90 days
- low-priority topics: every 180 days
- immediately after major WWDC / Xcode / Swift releases

## Freshness Metadata
Each reference should include:
- Status
- Last reviewed
- Confidence
- Official sources
- Related skills

## Refresh Trigger
Refresh a reference when:
- the official API surface changed,
- the recommended baseline changed,
- a major migration path changed,
- a currently recommended third-party library shipped a breaking change.
