# iOS Discovery

Use this skill during the **需求 / discovery** workflow.

## Default Principles
- Keep the user in the loop.
- Clarify the problem before expanding the feature set.
- Define the MVP boundary early.
- Surface policy, permissions, monetization, and feasibility risks before implementation.
- Avoid turning discovery into architecture prematurely.

## Required Output
1. Problem statement
2. Target user
3. Core use cases
4. MVP boundary
5. Feasibility / policy / product risks
6. Recommended next step
