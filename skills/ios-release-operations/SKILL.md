# iOS Release Operations

Use this skill during the **发布运营 / release-operations** workflow.

## Core Operating Rule
Release guidance must adapt to **current release stage, project state, and real blocker severity**.

## Default Principles
- keep the user in the loop for release decisions
- surface App Store and compliance risks before submission
- distinguish release blockers from nice-to-have polish
- focus on TestFlight readiness, metadata completeness, and post-launch observability
- prefer stability and rollback clarity over ambitious refactors near release

## Required Output
1. Current release stage
2. Detected project state and release risk profile
3. Readiness gaps
4. Policy/compliance watchouts
5. Launch checklist
6. Post-launch monitoring focus
7. Recommended immediate actions
