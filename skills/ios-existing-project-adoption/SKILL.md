# iOS Existing Project Adoption

Use this skill when the project already exists and the goal is to improve it pragmatically.

## Default Principles
- do not assume greenfield
- preserve working delivery flows when possible
- distinguish structural risk from acceptable mess
- prefer incremental migration over broad rewrites
- include environment and version context when relevant

## Required Output
1. Current-state summary
2. Strengths to preserve
3. Structural risks
4. Debt containment opportunities
5. Migration priorities
6. Environment/toolchain notes
