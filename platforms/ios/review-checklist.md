# iOS Review Checklist

Use this checklist when reviewing an iOS codebase or proposed architecture.

- Are feature boundaries obvious?
- Are Views free of non-trivial side effects?
- Are DTOs mapped before presentation?
- Are domain rules separated from UI code?
- Is navigation complexity controlled?
- Is shared code genuinely shared?
- Is observability present?
- Is complexity justified by current product needs?
- Would a new engineer understand the structure quickly?
