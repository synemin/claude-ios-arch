# iOS Discovery Checklist

Use this checklist before app bootstrap.

## Problem
- What user problem is being solved?
- Why should this be an iOS app?
- What existing alternatives already solve part of it?

## User
- Who is the primary user?
- What situation triggers usage?
- What counts as first value delivered?

## MVP
- What is the smallest end-to-end user journey worth shipping?
- What is explicitly out of scope?
- What dependencies could block MVP delivery?

## Risks
- policy / App Store review risk
- permissions / privacy risk
- monetization / IAP complexity
- backend / model dependency risk
- retention risk if first value is delayed
