# End-to-End Example: Solo Founder Journey

## Discovery
Use `ios-discovery` to define:
- target user
- MVP boundary
- key App Store / privacy risks

## Development
Use:
- `ios-app-bootstrap` for the app baseline
- `ios-feature-bootstrap` for first core feature
- `ios-testing-strategy` for minimal confidence plan
- `ios-tooling-decision` for lightweight tool choices
- `ios-debt-check` before beta

## Release & Operations
Use `ios-release-operations` to prepare:
- TestFlight readiness
- App Store metadata
- submission checklist
- launch monitoring priorities

## Principle
Keep the workflow light, practical, and biased toward shipping without hidden risk.
